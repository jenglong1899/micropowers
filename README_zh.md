# micropowers

本技能合集并非为了快速产出代码而设计，而是为了代码质量而设计，实践证明 AI 写的代码必须被认真审查。

- AGENTS.md（放在~/.codex）: 包含了记忆机制。codex有记忆机制，为什么用我这个？我看过.codex目录里面的记忆，我对其不太满意。你必须阅读这个文档才能使用，因为这个目前还是半自动的
    - 你要在项目根目录下的 AGENTS.md 中维护关于这个项目的记忆，比如架构是怎样的，这块不要偷懒，你要自己先写下大概的架构，然后剩下再让 AI 去补全。
    - 然后在项目根目录下的 AGENTS.md 的开头加上：
    - ```本文档是对代码库的一种摘要，目的是为了帮助 Agent 建立对代码库的理解，并不代表 Agent 不需要去阅读摘要所对应的代码文件，因为 Agent 有可能不知道“自己不知道”，比如 Agent 在实现某个功能的时候，做了某个兜底，但是由于没有看全代码，不知道原本的代码中已经有兜底了（不知道自己不知道），导致代码重复。```
- ~~plan-coding: 比codex自带的计划模式更好用。~~ 不知道GPT5.5是咋回事，感觉变笨了，花费时间制定了一套计划，结果后面做的又是不一样的。我现在改成了用GPT5.2，好多了，我新写了一个 simple-plan-coding，先让AI写个简单的计划（基本一分钟就能看完），然后告诉它一次只写100行以内的代码，一边写一边纠正。
- incremental-coding: 大部分情况下都应该用simple-plan-coding，而不是这个skill。一次只写100行以内的代码，并且在写之前简要说明要做的事情。这样顺应人性，不然一次性输出四五百行代码，人类是懒得看的。
