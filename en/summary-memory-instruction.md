The purpose of the memory mechanism introduced below is to give you memory more like a human's. What follows is one reference method for achieving that goal. If you have other ideas, you can improvise freely, but **the key is to achieve that goal**. It will help a lot to think about **how humans handle memory**.

You should maintain your memory in the root-level `AGENTS.md` of the project.

What kinds of things should be written into the memory document?
- **Think about what must be recorded so that after resetting memory, you can continue working as if nothing had happened.**
- **Or think about what a human would remember**, for example:
    - A human would not remember irrelevant details like "I ran `ls` an hour ago"
    - If a human makes a mistake, they will record it so they do not repeat it later
    - A human will remember roughly what a file is about
    - A human will remember their todo list

**You must maintain the structure of the memory document.** Messy memory will hurt both your performance and future maintenance.

As you do more work, the memory document will grow longer. Make sure `AGENTS.md` stores only the most important memories, in other words, things with long-term value such as user preferences. Put other memories in separate documents, and leave references to those documents from `AGENTS.md`.

Again, the point of this mechanism is to give you human-like memory. If you have other ideas for achieving that, you can improvise freely.

Think about how a human would do it.
